GitHub

GitHub is a platform for developers that is driven by AI power and brings convenience in creating, storing, managing and sharing code. GitHub uses Git, a software that helps to manage Git repositories. Git, is simply a version control system to manage and keep track of all the updates you do in your source code. GitHub manages Git repositories. It is subsidiary of Microsoft since 2018.Github platform was developed on October 19, 2007.Github has an annual global event called GitHub Universe.


GitHub has a simple yet professional Sign in page that allows its users to sign into its platform, it is an import part for them to authenticate an account. In their primary sign in they have two labels, the "Username or email address" and the "Password" along with "Forgot password?" beside it and a green button representing Sign in. Above the form, they contain text "Sign in to GitHub" with their logo above it and they also have an alternative of signing in with a passkey and a new account creation section below the main form.


What is Similar?

Both of our forms take input from the user and is user-friendly.

Both of our forms display an error message if the input is invalid or incorrect.

Both of our forms are aligned at the center of the page and have similar overall surface structure.

Both of our forms contain a button to submit/sign in accordingly.


What is Different ?

Our form contains a message section with placeholder, which is not contained by GitHub.

GitHub contains Forgot password section which upon clicking redirects to reset password section also containing CAPTCHA resembling test to confirm we are not robot and this we do not contain.

On invalid input we display an alert message at the center whereas GitHub in the same scenario displays a styled invalid text just above the Username input section.

On clicking Create account section, GitHub redirects us to another page.
